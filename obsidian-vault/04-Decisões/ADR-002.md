# ADR-002: MVP começa por aviamentos, não tecidos

Espelho de `docs/decisions/0002-mvp-aviamentos-primeiro.md` (fonte de verdade).

**Status:** Aceito · **Data:** 2026-05-26 · **Decisores:** Diego

## Contexto

Proposta original focava em **tecidos** (largura, composição, gramatura, unidade de venda
variável — metro/rolo/peça), complicando modelagem e prompt de IA. Diego decidiu pivotar para
**aviamentos** (linha, botão, fita) primeiro.

## Decisão

MVP foca exclusivamente em aviamentos. Tecidos ficam para uma fase posterior. Ver [[Glossário]]
(termo "Aviamento").

## Alternativas consideradas

- Suportar tecidos e aviamentos desde o início — rejeitada, complexidade prematura
- Começar por tecidos (proposta original) — rejeitada, unidade de venda variável e atributos
  têxteis do ML são numerosos e cheios de regras especiais
- **Aviamentos primeiro (escolhida)** — atributos mais simples, variação principal é só cor

## Consequências

**Boas:** pipeline mais simples no MVP; variação principal só cor (caso limpo para validar
atribuição via [[IA|Vision]]); menos edge cases iniciais.

**Tradeoffs:** catálogo inicial cobre só uma fatia do catálogo total da empresa; versão para
tecidos vai exigir reabertura do modelo de dados (campos opcionais, não breaking change).

**Como reverter:** decisão de produto — expansível a qualquer momento sem refactor estrutural,
adicionando campos opcionais em `familias` e ajustando o prompt de IA por categoria.
